# Handoff Documents

This directory stores handoff notes for future Codex sessions.

## Naming

- Use `YYYY-MM-DD-short-topic.md`.
- Keep one handoff per continuation topic.

## Content

- Summarize only the current state and next actions.
- Reference specs, plans, diffs, and code paths instead of duplicating them.
- Redact secrets, tokens, credentials, and personal data.

## Cleanup

- Remove handoff files after their follow-up work is completed or committed.
- Keep this README as the directory convention.
